# Island Manila Bites Demo Website

## What This Demo Is

Island Manila Bites is a standalone local demo website for a fictional Brampton food business offering Caribbean and Filipino-inspired snack boxes, sweets, party trays, and weekly preorder pickups.

The customer-facing page is written to feel like a real preorder website. The fictional/demo context is kept to this README and the small footer disclaimer.

Demo site by Tristan Web Studio. Island Manila Bites is a fictional sample brand created to show a preorder website concept for local food businesses. Real food businesses should confirm local health, zoning, insurance, and food safety requirements before selling.

## Target Business Type

This concept is for home cooks, bakers, snack sellers, caterers, and Instagram-based food businesses that take weekly preorders, party tray requests, and catering inquiries for family gatherings, birthdays, office lunches, community events, and weekend orders.

## Portfolio Demo Strategy Angle

The demo shows how Tristan Web Studio can help local food businesses replace scattered Instagram DMs, WhatsApp messages, and manual preorder lists with a clearer customer flow:

- Weekly menu cards
- Snack box and party tray packages
- Social proof-style trust cards
- Pickup and service area details
- Preorder expectations before the form
- A structured preorder and catering inquiry form
- FAQs that reduce repeated questions about pricing, deposits, spice level, pickup, delivery requests, and allergies

The business problem solved is simple: customers can send clearer preorder and catering details up front, which means fewer back-and-forth messages and easier quoting for the business owner.

## Static Setup Only

This project is plain static HTML, CSS, and JavaScript. It does not require npm, a framework, a build tool, a backend, a database, or a payment processor.

## Files Included

- `index.html` — page content and structure
- `styles.css` — responsive layout, visual styling, and CSS illustrations
- `main.js` — static form validation, date minimum, success message, and reset behavior
- `README.md` — project notes and portfolio positioning

## Technical Notes

The preorder form is front-end only for demonstration. It validates required fields, shows helpful messages, and does not send data to a backend.

No copyrighted food photography or external libraries are used. The food visuals are built with CSS illustration styles.
